How To Install ERPNext Version 15 In Ubuntu 22.04


In this guide, we'll walk through the process of installing ERPNext version 15 on Ubuntu 22.04, step by step, making it easy for you to set up your environment.

Prerequisites:

Before we begin, ensure you have the following in place for optimal functionality:

System Requirements:

  • Updated Ubuntu 22.04
  • A user with sudo privileges

Hardware Requirements:

  • 4GB RAM
  • 40GB Hard Disk

Pre-requisites:

  • Python 3.10 or Python 3.11
  • Node.js 18
  • MariaDB 10.6.6+

    • Common for all Frappe versions:

  • Redis 6 (caching and realtime updates)
  • yarn 1.12+ (js dependency manager)
  • pip 20+ (py dependency manager)
  • wkhtmltopdf (version 0.12.5 with patched qt) (for pdf generation)
  • cron (bench's scheduled jobs: automated certificate renewal, scheduled backups)
  • NGINX (proxying multitenant sites in production)

Step 1: Update and Upgrade Packages

First, let's ensure our system is up to date by running the following commands:

 
sudo apt-get update -y && sudo apt-get upgrade -y

Step 2: Create a New User

To maintain security, let's create a new user that we'll use for running ERPNext:

 
sudo adduser [frappe-user]
usermod -aG sudo [frappe-user]
su [frappe-user]
cd /home/[frappe-user]

Replace [frappe-user] with your desired username.

Step 3: Install Required Packages

Now, let's install the necessary packages for ERPNext:

  • Install GIT:
    •  
    sudo apt-get install git
  • Install Python:
    • sudo apt-get install python3-dev python3.10-dev python3-setuptools python3-pip python3-distutils
  • Install Python Virtual Environment:
    • sudo apt-get install python3.10-venv
  • Install Software Properties Common:
    • sudo apt-get install software-properties-common
  • Install MariaDB:
    • sudo apt install mariadb-server mariadb-client
  • Install Redis Server:
    • sudo apt-get install redis-server
  • Install other necessary packages:
    • sudo apt-get install xvfb libfontconfig wkhtmltopdf libmysqlclient-dev

    Step 4: Configure MySQL Server

    Setup the server and configure MySQL by running:

     
    sudo mysql_secure_installation

    Follow the prompts to set up the MySQL server.

    Step 5: Install CURL, Node, NPM, and Yarn

  • Install CURL:
    •  
    sudo apt install curl
  • Install Node:
    • curl https://raw.githubusercontent.com/creationix/nvm/master/install.sh | bash
source ~/.profile
nvm install 18
  • Install NPM & Yarn:
    • sudo apt-get install npm && sudo npm install -g yarn

    Step 6: Install Frappe Bench

    Install Frappe Bench using pip3:

     
    sudo pip3 install frappe-bench

    Step 7: Initialize Frappe Bench

    Initialize Frappe Bench with ERPNext version 15:

     
    bench init --frappe-branch version-15 frappe-bench
cd frappe-bench

    Step 8: Change User Directory Permissions

    Ensure the bench user has execution permission to the home directory:

     
    chmod -R o+rx /home/[frappe-user]

    Step 9: Create a New Site

    Create a new site for ERPNext:

     
    bench new-site [site-name.com]

    Step 10: Install ERPNext, HRMS and other Apps

  • Download and install ERPNext and other desired apps:
    •  
    bench get-app --branch version-15 payments
bench get-app --branch version-15 erpnext
bench get-app --branch version-15 hrms
  • Install India Compliance (For India Region):
    • bench get-app --branch version-15 https://github.com/resilient-tech/india-compliance.git

    Step 11: Start ERPNext

    Start the ERPNext server:

     
    bench start

    You can access ERPNext at [YOUR SERVER IP:8000] OR 127.0.0.1:8000 OR 0.0.0.0:8000

    Step 12: Setting ERPNext for Production

  • Enable Scheduler & Disable maintenance mode:
    •  
    bench --site [site-name.com] enable-scheduler && bench --site [site-name.com] set-maintenance-mode off
  • Enable Developer mode (If needed):
    • bench --site [site-name.com] set-config -g developer_mode 1
  • Setup production config:
    • sudo bench setup production [frappe-user]
  • Setup NGINX:
    • bench setup nginx
  • Restart Supervisor and Launch Production Mode:
    • sudo supervisorctl restart all
sudo bench setup production [frappe-user]

    Your ERPNext instance is now in production mode and accessible via your IP address.
